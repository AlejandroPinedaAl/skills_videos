# Faceless Benefit Stack

## Framework Logic
No face does not mean no personality. This format uses visual rhythm, strong copywriting, and strategic evidence (screenshots, dashboards, numbers) to build desire without a spokesperson. It's the highest-scalability format because it doesn't require talent scheduling or filming sessions.

**Best use case for SaleADS:** Platform demos, feature showcases, ROAS/result proofs, benefit comparisons.

---

## Visual Structure (5-plane system)

```
PLANE 1 — HEADLINE (0-2s)
├── Full-screen text — large, bold, one idea
├── Background: solid color, gradient, or slow motion b-roll
├── This IS the hook — must stop the scroll as a text card alone
└── Examples:
    "¿Cuánto dinero llevas gastando en ads sin saber si funciona?"
    "El 80% de los negocios no optimiza sus campañas en tiempo real."
    "Tus competidores ya usan IA para sus anuncios."

PLANE 2 — PROBLEM VISUAL (2-5s)
├── Show the pain without narration
├── Options: spreadsheet chaos, manual process, confusing dashboard, wasted budget notification
├── On-screen text names the problem
└── Minimal audio: ambient sound or music only

PLANE 3 — SOLUTION VISUAL (5-8s)
├── Product/platform in action — the actual UI, the actual result
├── Screen recording is ideal — real product beats mockup
├── Highlight the key action with zoom or arrow annotation
└── On-screen text: "[Product] makes [problem] disappear"

PLANE 4 — PROOF (8-11s)
├── Evidence layer — one of: number/metric, testimonial text overlay, comparison
├── Keep it specific: "ROAS 4.7x" not "great results"
├── Visual: result dashboard, before/after metric, customer quote card
└── On-screen text: the proof claim

PLANE 5 — CTA (11-15s)
├── Offer + action — one sentence max
├── Create reason to act now
└── On-screen text: CTA bold + offer detail below
```

---

## Benefit Stack Logic

A "benefit stack" means listing multiple benefits in rapid succession. The key is sequencing them by emotional weight:

1. **Lead with time or money saved** (most universal)
2. **Follow with ease/simplification** (reduces friction)
3. **Close with outcome/transformation** (desire driver)

Example stack for SaleADS:
1. "Sin contratar diseñadores ni copywriters"
2. "Sin aprender plataformas complicadas"
3. "Con campañas que aprenden y mejoran solas"

**Anti-pattern:** Don't stack features. Stack outcomes.
- ❌ "Generación de copy, A/B testing, dashboard en tiempo real, integración con Meta"
- ✅ "Menos tiempo configurando. Más leads llegando. Más certeza en cada peso invertido."

## Text Animation Styles (Visual Direction)

| Style | When to Use |
|---|---|
| Pop-in (fast appear) | Energetic, young audience, TikTok |
| Slide-in from left | Professional, B2B, Meta Feed |
| Typewriter | High-emphasis single claim, YouTube |
| Kinetic (word by word) | Emotional copy, storytelling beats |
| Static bold card | Faceless product, bold claims |

---

## Audio for Faceless Videos
- **Background music:** low-energy for serious B2B, mid-energy for product/consumer
- **No voiceover needed** but optional: AI voice or text-to-speech acceptable
- **Sound design:** subtle UI sounds (click, notification, success chime) for software demos
- **Captions:** still required even without a speaker

---

## Adaptation by Platform

| Platform | Key Adjustment |
|---|---|
| TikTok | Add faster cuts (0.8-1s per plane), music mandatory |
| Meta Feed | Can run 15-30s, text must be readable on desktop |
| Meta Stories | All text in top 70% of frame, large CTA bottom |
| Google Display | First 5s must carry full message (some users skip) |
| YouTube Pre-roll | Clear value prop before 5s skip button |

## Script Output Template

```
[TITLE]: Faceless – [Business Name] – [Platform] – [Duration]s

PLANE 1 / HEADLINE (0-2s):
TEXT: "[full-screen headline]"
BACKGROUND: [color / b-roll description]
ANIMATION: [pop-in / slide / static]

PLANE 2 / PROBLEM (2-5s):
VISUAL: [what to show]
TEXT OVERLAY: "[problem label]"
AUDIO: [music note / sound design]

PLANE 3 / SOLUTION (5-8s):
VISUAL: [screen recording or product shot description]
TEXT OVERLAY: "[solution claim]"
HIGHLIGHT: [what to annotate or zoom]

PLANE 4 / PROOF (8-11s):
VISUAL: [metric / quote card / comparison]
TEXT: "[specific proof claim with number]"

PLANE 5 / CTA (11-15s):
TEXT: "[offer + action]"
SUBTEXT: "[supporting detail or urgency]"
BUTTON: "[CTA text]"
```
